# 09.tool-03 AI 记忆体系管理入口

> Current spec: .ai/L3#Standards/standards/09.tool-03.ai-gogogo.md

## 两个入口

```bash
./gogogo.sh 8f    # 建立所有映射（最常用）
./gogogo.sh 8g    # 移除所有映射
./gogogo.sh 8     # 映射管理菜单
./gogogo.sh ai    # AI 体系管理菜单
```

## 映射层级

| 层级 | 源 | 目标 | 方式 |
|------|-----|------|------|
| L1#项目概览 (Overview) | `.ai/L1#Overview/guide.md` | `AGENTS.md` `CLAUDE.md` `.cursorrules` `.windsurfrules` | 单文件链接 |
| L2#规范索引 (Index) | `.ai/L2#Index/{README,toc}.md` | `.kiro/steering/` `.cursor/rules/` `.claude/memories/` `.continue/rules/` `.windsurf/rules/` | 逐文件链接 |
| L3#完整规范 (Standards) | `.ai/L3#Standards/standards/` | 各 IDE 的 `zero-standards` 按需规则目录 | 整目录链接（可选） |

**注意**：L0#工作执行 (Execution) 的 hooks/skills/specs 是源文件层，项目使用时可扩展映射到具体 IDE。

## 核心规则

- **逐文件链接**，不整目录链接（避免 README.md 污染上下文）
- **L2 自动载入，L3 按需读取**（L3 通过 globs 或人工检索触发）
- **禁止手动 `ln -sf`**，必须通过 `./gogogo.sh 8f`
- `gogogo.ai.sh` 双重角色：函数库 + 条件菜单（`BASH_SOURCE` 判断）

## 禁止

- `.ai/` 内放运维脚本
- 把 L3#完整规范 (Standards) 映射成旧 `condensed` 目录
- 手动建立映射
