## 核心理念

**文件系统优先**：目录存在即显示，数据库配置是增强不是必需。

```
文件系统扫描 → manifest_list 匹配 → 警告标记 → 缓存 → 渲染
```

---

## 显示规则

```javascript
// 显示名称：有 shortName 用 shortName，没有就用目录名
const displayName = item.shortName || item.folderName

// 无硬编码默认值
icon: dbRecord?.icon || null      // 没有就是 null
color: dbRecord?.color || null    // 前端根据 null 显示警告
```

---

## 警告状态

| 状态 | 说明 | 显示效果 |
|------|------|---------|
| `NO_DB_RECORD` | 数据库中无此记录 | 显示目录名 + 红色警告 |
| `MISSING_FIELD` | 有记录但字段不完整 | 显示已有值 + 黄色警告 |
| 无警告 | 配置完整 | 正常显示 |

### 显示效果示例

```
✅ 用户管理          （配置完整）
⚠️ settings          （无数据库记录，显示目录名）
⚠️ 权限配置          （缺少字段：icon、color）
```

## 警告检查

```javascript
function checkWarnings(dbRecord) {
  const warnings = []

  if (!dbRecord) {
    warnings.push({ type: 'NO_DB_RECORD', message: '数据库中无此记录' })
    return warnings
  }

  if (!dbRecord.shortName) warnings.push({ field: 'shortName' })
  if (!dbRecord.icon) warnings.push({ field: 'icon' })
  if (!dbRecord.color) warnings.push({ field: 'color' })
  if (dbRecord.sortOrder == null) warnings.push({ field: 'sortOrder' })

  return warnings
}
```

## 排序规则

```javascript
// 有 sortOrder 的按数值排，没有的按目录名字母序
items.sort((a, b) => {
  if (a.sortOrder !== null && b.sortOrder !== null) {
    return a.sortOrder - b.sortOrder
  }
  if (a.sortOrder !== null) return -1
  if (b.sortOrder !== null) return 1
  return a.key.localeCompare(b.key)
})
```
