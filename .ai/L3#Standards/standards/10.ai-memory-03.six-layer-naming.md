# 10.ai-memory-03.six-layer-naming - 六层架构官方命名对照表

> **文档版本**：v1.0.0
> **创建日期**：2026-04-17
> **来源项目**：AiLinkDog
> **定位**：定义六层 AI 记忆体系的官方中英文命名规范

---

## 概述

本文档定义了**六层 AI 记忆体系**的官方命名对照表，包括中文名称、英文名称、文件夹命名格式和文档引用格式。

**适用场景**：
- 从七层架构迁移到六层架构的项目
- 需要统一中英文命名的项目
- 需要跨平台兼容的文件夹命名

---

## 六层 vs 七层架构

### 层级对应关系

| 六层架构 | 七层架构 | 说明 |
|---------|---------|------|
| L0#工作执行 (Execution) | L0 | 项目进度管理 |
| L1#项目概览 (Overview) | L1 | 项目概览 |
| L2#规范索引 (Index) | L2 | 规范索引 |
| L3#完整规范 (Standards) | L3 + L4 | 合并规范层 |
| L4#操作日志 (Changelog) | L5 | 操作日志 |
| L5#知识图谱 (Knowledge) | L6 | 知识图谱 |

**核心变化**：
- ✅ 七层的 L3（精炼规范）+ L4（完整规范）→ 六层的 L3（完整规范）
- ✅ 七层的 L5（操作日志）→ 六层的 L4（操作日志）
- ✅ 七层的 L6（知识图谱）→ 六层的 L5（知识图谱）

## 使用规范

### 1. 文件夹命名（物理目录）

**格式**：`.ai/L#英文名/`

**示例**：
```bash
.ai/L0#Execution/
.ai/L1#Overview/
.ai/L2#Index/
.ai/L3#Standards/
.ai/L4#Changelog/
.ai/L5#Knowledge/
```

**规则**：
- ✅ 必须使用 `L#英文名/` 格式
- ✅ 英文名首字母大写
- ✅ 使用 `#` 连接层级和英文名
- ❌ 不使用中文文件夹名（避免跨平台兼容性问题）
- ❌ 不使用下划线或连字符（如 `L0_Execution` 或 `L0-Execution`）

**理由**：
- ✅ 避免跨平台文件系统兼容性问题
- ✅ 符合国际化项目的最佳实践
- ✅ 便于脚本和工具处理
- ✅ Git 和 IDE 更友好
- ✅ `#` 符号在文件系统中合法且直观

---

### 2. 文档中引用（完整格式）

**格式**：`L#中文名 (英文名)`

**示例**：
```markdown
L0#工作执行 (Execution)
L1#项目概览 (Overview)
L2#规范索引 (Index)
L3#完整规范 (Standards)
L4#操作日志 (Changelog)
L5#知识图谱 (Knowledge)
```

**规则**：
- ✅ 规范文档中必须使用完整格式
- ✅ 中文名和英文名都要包含
- ✅ 英文名放在括号中
- ✅ 便于 AI 理解和人类阅读

**理由**：
- ✅ 中文名便于人类理解
- ✅ 英文名便于 AI 理解
- ✅ 两者结合，最大化可读性
- ✅ 明确对应关系

---

### 3. 日常沟通（简称）

**格式**：`L0`、`L1`、`L2`、`L3`、`L4`、`L5`

**示例**：
```
"请查看 L2 找到 API 设计规范"
"更新 L3 中的错误处理规范"
"这个决策记录在 L4 中"
```

**规则**：
- ✅ 日常对话可以简称
- ✅ AI 会自动关联到完整命名
- ✅ 提高沟通效率

---

### 4. 路径引用（代码/脚本）

**格式**：`.ai/L#英文名/`

**示例**：
```bash
SOURCE_DIR=".ai/L2#Index"
cd ".ai/L3#Standards/standards"
ln -s "../../.ai/L2#Index/toc.md"
```

**规则**：
- ✅ 脚本中使用完整路径
- ✅ 使用双引号包裹（因为包含 `#` 字符）
- ✅ 确保跨平台兼容性

## 设计决策

### 决策 1：文件夹使用英文名

**选择**：`.ai/L0#Execution/`（而非 `.ai/L0#工作执行/`）

**理由**：
- ✅ 避免跨平台文件系统兼容性问题
- ✅ 符合国际化项目的最佳实践
- ✅ 便于脚本和工具处理
- ✅ Git 和 IDE 更友好

---

### 决策 2：文档引用使用中英文混合

**选择**：`L0#工作执行 (Execution)`

**理由**：
- ✅ 中文名便于人类理解
- ✅ 英文名便于 AI 理解
- ✅ 两者结合，最大化可读性
- ✅ 明确对应关系

---

### 决策 3：英文命名选择

**L0#Execution**（工作执行）：
- 强调"执行层"的概念
- 包含 hooks/skills/specs/workflows

**L1#Overview**（项目概览）：
- 最常见的"概览"术语
- 提供项目快速了解

**L2#Index**（规范索引）：
- 简洁明确的"索引"概念
- 符合目录索引的定位

**L3#Standards**（完整规范）：
- 业界通用的"规范"术语
- 强调标准化和规范性

**L4#Changelog**（操作日志）：
- 开发中最常见的"变更日志"术语
- 记录操作历史

**L5#Knowledge**（知识图谱）：
- 涵盖知识图谱的概念
- 构建关系网络

---

### 决策 4：使用 `#` 而非 `-` 或 `_`

**选择**：`L0#Execution`（而非 `L0-Execution` 或 `L0_Execution`）

**理由**：
- ✅ `#` 在文件系统中合法
- ✅ 视觉上更清晰地分隔层级和名称
- ✅ 不与连字符命名规范冲突（如 `api-design.md`）
- ✅ 不与下划线命名规范冲突（如 `admin_users` 表名）
- ✅ 符号语义明确（`#` 常用于标签、标识符）

## 从七层迁移到六层

### 迁移步骤

**Phase 1: 重命名文件夹**
```bash
# 备份
cp -r .ai .ai.backup

# 重命名（如果从七层迁移）
mv .ai/L0 ".ai/L0#Execution"
mv .ai/L1 ".ai/L1#Overview"
mv .ai/L2 ".ai/L2#Index"

# 合并 L3 和 L4（七层）→ L3（六层）
mkdir -p ".ai/L3#Standards"
mv .ai/L3/* ".ai/L3#Standards/"
mv .ai/L4/standards ".ai/L3#Standards/"
mv .ai/L4/practices ".ai/L3#Standards/"
mv .ai/L4/cases ".ai/L3#Standards/"
# ... 其他子目录

# 重命名 L5 → L4
mv .ai/L5 ".ai/L4#Changelog"

# 重命名 L6 → L5
mv .ai/L6 ".ai/L5#Knowledge"
```

**Phase 2: 更新文档引用**
```bash
# 批量替换文档中的引用
find ".ai/L3#Standards" -name "*.md" -exec sed -i '' 's/L0#工作执行/L0#工作执行 (Execution)/g' {} \;
find ".ai/L3#Standards" -name "*.md" -exec sed -i '' 's/L1#项目概览/L1#项目概览 (Overview)/g' {} \;
find ".ai/L3#Standards" -name "*.md" -exec sed -i '' 's/L2#规范索引/L2#规范索引 (Index)/g' {} \;
find ".ai/L3#Standards" -name "*.md" -exec sed -i '' 's/L3#完整规范/L3#完整规范 (Standards)/g' {} \;
find ".ai/L3#Standards" -name "*.md" -exec sed -i '' 's/L4#操作日志/L4#操作日志 (Changelog)/g' {} \;
find ".ai/L3#Standards" -name "*.md" -exec sed -i '' 's/L5#知识图谱/L5#知识图谱 (Knowledge)/g' {} \;
```

**Phase 3: 更新脚本路径**
```bash
# 更新 sync.sh 或 gogogo.sh 中的路径
sed -i '' 's|.ai/L2|.ai/L2#Index|g' sync.sh
sed -i '' 's|.ai/L3|.ai/L3#Standards|g' sync.sh
```

**Phase 4: 重建 symlink**
```bash
# 运行同步脚本
./sync.sh rebuild
# 或
./gogogo.sh ai
```

**Phase 5: 验证**
```bash
# 检查文件夹结构
tree -L 1 .ai/

# 检查 symlink
ls -la .kiro/steering/
ls -la .claude/rules/

# 检查文档引用
grep -r "L0#工作执行 (Execution)" .ai/L3#Standards/
```

## 零容忍规则

### 文件夹命名规则

**规则 25**：**六层架构文件夹必须使用 `L#英文名/` 格式**（如 `.ai/L0#Execution/`）

**违规示例**：
- ❌ `.ai/L0/`
- ❌ `.ai/L0-Execution/`
- ❌ `.ai/L0_Execution/`
- ❌ `.ai/L0#工作执行/`

**正确示例**：
- ✅ `.ai/L0#Execution/`
- ✅ `.ai/L1#Overview/`
- ✅ `.ai/L2#Index/`

---

### 文档引用规则

**规则 26**：**六层架构文档引用必须使用中英文混合格式**（如 `L0#工作执行 (Execution)`）

**违规示例**：
- ❌ `L0#工作执行`
- ❌ `L0#Execution`
- ❌ `L0 工作执行`

**正确示例**：
- ✅ `L0#工作执行 (Execution)`
- ✅ `L1#项目概览 (Overview)`
- ✅ `L2#规范索引 (Index)`
