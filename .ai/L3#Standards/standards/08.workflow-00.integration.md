```
  └── ...

  ├── 技术栈规范

```

## 快速开始

### 1. 选择开发平台

| 平台 | 推荐指数 | 最适合 | 文档 |
|------|--------|--------|------|
| Cursor | ⭐⭐⭐⭐⭐ | 快速开发和原型 | 08.workflow-01 |
| Kiro | ⭐⭐⭐⭐⭐ | 深度思考和设计 | 08.workflow-02 |
| Claude Code CLI | ⭐⭐⭐⭐ | 自动化和 CI/CD | 08.workflow-03 |
| OpenCode | ⭐⭐⭐⭐ | 定制工作流 | 08.workflow-04 |

### 2. 配置环境

```bash
# 使用 ZERO 的 IDE 映射管理
./gogogo.sh 8f    # 映射到所有工具
./gogogo.sh 8a    # 仅映射到 Kiro
./gogogo.sh 8b    # 仅映射到 Cursor
```

### 3. 规范快速查询

**在 Cursor 中**
```
Cmd + K → "@rules 如何设计 API？"
```

**在 Kiro 中**
```
左侧面板 → Rules → 搜索功能名称
```

## 主要特性

### 1. 规范自动应用
- Tab 补全自动参考规范
- 代码审查自动应用规范检查

### 2. 工作日志自动化
- 自动生成日志文件名
- 插入标准模板
- 根据提交自动填充内容

### 3. 规范与日志闭环
```
查看规范 → 编码 → 记录日志 → 回顾 → 改进规范 → (循环)
```
