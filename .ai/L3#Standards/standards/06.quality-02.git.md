# 06.quality-02.git - Git 提交规范

> **当前规范**：`.ai/L3#Standards/standards/06.quality-02.git.md`

---

## Commit 格式

```
<type>(<scope>): <subject>
```

---

## 提交类型

| 类型 | 说明 |
|------|------|
| `feat` | 新功能 |
| `fix` | 修复 Bug |
| `docs` | 文档变更 |
| `style` | 代码格式 |
| `refactor` | 重构 |
| `perf` | 性能优化 |
| `test` | 测试 |
| `chore` | 杂项 |

---

## Subject 规则

- 祈使句，现在时态
- 首字母小写
- 不超过 50 字符
- 结尾不加句号

---

## 分支命名

| 类型 | 格式 |
|------|------|
| 功能 | `feature/<功能名>` |
| 修复 | `fix/<问题描述>` |
| 发布 | `release/<版本号>` |

---

## 举例

```
feat(user): 添加用户列表页面
fix(auth): 修复登录验证逻辑
refactor(api): 重构用户服务层
```
