# 06.quality-04.header - 文件头注释规范

> **当前规范**：`.ai/L3#Standards/standards/06.quality-04.header.md`

---

## 核心理念

**文件自包含文档化**：在文件开头用注释完整记录逻辑、思路、结构。

---

## 标准模板

```javascript
/**
 * ================================================================
 * 文件名: [完整文件名.扩展名]
 * 创建时间: YYYY-MM-DD
 * 更新时间: YYYY-MM-DD
 * ================================================================
 *
 * 【文件职责】
 * [1-3句话简述核心职责]
 *
 * 【核心架构】
 * 1. [架构点1]
 * 2. [架构点2]
 *
 * 【主要函数】
 * - functionName1(): 函数1的作用
 * - functionName2(): 函数2的作用
 *
 * 【依赖关系】
 * - 内部模块: @/utils/xxx
 * - 外部API: /api/v1/xxx
 *
 * 【更新日志】
 * - YYYY-MM-DD: [更新说明]
 * ================================================================
 */
```

---

## 必需字段

| 字段 | 说明 |
|------|------|
| 文件名 | 完整文件名 |
| 创建/更新时间 | YYYY-MM-DD |
| 【文件职责】 | 核心职责 |
| 【核心架构】 | 架构要点 |
| 【主要函数】 | 函数列表 |
| 【依赖关系】 | 依赖说明 |

---

## 同步规则

- 修改代码 → 更新注释
- 新增函数 → 更新【主要函数】
- 修改依赖 → 更新【依赖关系】
- 重大修改 → 更新【更新日志】
