> **文档版本**：v1.0.0
> **创建日期**：2025-01-12

## 📍 文档定位

本文档定义 ZERO 框架的**命名哲学**，是所有命名规范的指导原则。

---

## 🎯 核心哲学

### 完整描述 > 缩写简写

```
❌ cmd, cfg, usr, btn, msg, req, res
✅ command, config, user, button, message, request, response

❌ auth.cmd.php
✅ authentication.api.php

❌ usr_cfg_lvl
✅ user_config_level
```

**原因**：
- AI 模型对完整单词的语义理解更准确
- 减少歧义（`cmd` 是 command? commander? 还是其他？）
- 新开发者无需查阅缩写表

### AI 友好 > 人类打字方便

```
❌ 为了少打几个字而缩写
✅ 为了 AI 准确理解而完整描述

❌ getUserInfo()
✅ getUserProfileInformation()

❌ handleErr()
✅ handleErrorResponse()
```

**原因**：
- AI 开发时代，代码由 AI 生成，人类审核
- 完整命名让 AI 生成的代码更准确
- IDE 自动补全让长命名不再是负担

### 行业通用 > 项目特定

```
❌ 项目特定缩写：.cmd.php, .lab.php, .open.php
✅ 行业通用命名：.api.php, .controller.php, .service.php

❌ 自创术语：manifest_list, cmder_session
✅ 行业术语：page_config, admin_session
```

**原因**：
- AI 训练数据中行业通用术语出现频率更高
- 新团队成员无需学习项目特定术语
- 跨项目复用代码更容易

### 精准语义 > 模糊简称

```
❌ data, info, item, obj, temp
✅ userProfileData, orderDetailInformation, productListItem

❌ process(), handle(), do()
✅ processPaymentTransaction(), handleUserAuthentication()

❌ utils.js, helpers.js, common.js
✅ stringFormatUtils.js, dateTimeHelpers.js, validationCommon.js
```

**原因**：
- 精准命名是自文档化的最佳实践
- 减少代码阅读时的认知负担
- AI 能更准确理解代码意图

## 📋 命名长度指南

### 可接受的长度

| 类型 | 推荐长度 | 举例 |
|------|---------|------|
| 变量名 | 10-30 字符 | `userProfileInformation` |
| 函数名 | 15-40 字符 | `validateUserEmailAddress()` |
| 文件名 | 15-50 字符 | `UserProfileSettings.view.vue` |
| 数据库表 | 20-50 字符 | `user_profile_settings_config` |
| 数据库字段 | 15-40 字符 | `base_profile_email_verified` |

### 长度 vs 清晰度

```
✅ 宁可长一点，也要清晰
getUserAuthenticationTokenFromLocalStorage()

❌ 不要为了短而牺牲清晰度
getAuthTkn()
```

## 🔄 文件后缀规范

### 行业通用后缀

| 后缀 | 用途 | 举例 |
|------|------|------|
| `.api.php` | API 入口文件 | `user.api.php` |
| `.controller.php` | 控制器 | `UserController.php` |
| `.service.php` | 服务层 | `UserService.php` |
| `.model.php` | 数据模型 | `UserModel.php` |
| `.repository.php` | 数据仓库 | `UserRepository.php` |
| `.helper.php` | 辅助函数 | `StringHelper.php` |
| `.config.php` | 配置文件 | `database.config.php` |
| `.include.php` | 包含文件 | `bootstrap.include.php` |

### 禁止使用的后缀

| 禁止 | 原因 |
|------|------|
| `.cmd.php` | 项目特定，语义不明 |
| `.lab.php` | 项目特定，语义不明 |
| `.open.php` | 项目特定，语义不明 |
| `.inc.php` | 缩写，应用 `.include.php` |

## 💡 AI 开发友好性检查

### 命名自检问题

1. **AI 能否仅通过名称理解用途？**
   - ❌ `processData()` - 处理什么数据？
   - ✅ `processUserRegistrationData()` - 清晰

2. **是否使用了行业通用术语？**
   - ❌ `cmder_session` - 什么是 cmder？
   - ✅ `admin_session` - 行业通用

3. **是否避免了项目特定缩写？**
   - ❌ `qy_usr_cfg` - 需要查阅项目文档
   - ✅ `quietyou_user_config` - 自解释

4. **新开发者能否立即理解？**
   - ❌ `handleCb()` - 需要上下文
   - ✅ `handleApiResponseCallback()` - 自解释

## 🚫 UI 设计规范

### 默认 UI 框架

ZERO 框架默认采用 **TDesign** 作为 UI 组件库：

- **Vue 3 项目**：`tdesign-vue-next`（PC端）、`tdesign-mobile-vue`（移动端）
- **React 项目**：`tdesign-react`（PC端）、`tdesign-mobile-react`（移动端）
- **图标库**：`tdesign-icons-vue-next` / `tdesign-icons-react`

```vue
<!-- TDesign 图标使用示例 -->
<template>
  <t-icon name="check-circle" />
  <t-icon name="user" size="24px" />
  <t-icon name="setting" color="#07c160" />
</template>
```

### 禁止使用 Emoji

在正式的产品界面中，**禁止使用 Emoji 作为图标或装饰**。

```
❌ 🏆 专业认证
✅ 专业认证（配合 TDesign Icon）

❌ 📱 下载 App
✅ 下载 App（配合 <t-icon name="download" />）

❌ ✅ 操作成功
✅ 操作成功（配合 <t-icon name="check-circle" />）
```

**原因**：
- Emoji 在不同系统/浏览器显示效果不一致
- Emoji 风格过于卡通，不够专业
- Emoji 无法自定义颜色和大小
- 正式产品应使用 TDesign Icons 保持风格统一

**例外情况**：
- 用户生成内容（聊天、评论等）
- 非正式的内部工具
- 开发文档中的标记（如本文档）
