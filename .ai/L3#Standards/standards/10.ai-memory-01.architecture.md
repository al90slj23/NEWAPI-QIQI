## 六层定义

| 层级 | 名称 | 物理目录 | 职责 |
|------|------|----------|------|
| L0 | L0#工作执行 (Execution) | `.ai/L0#Execution/` | hooks、skills、specs、templates、workflows |
| L1 | L1#项目概览 (Overview) | `.ai/L1#Overview/` | 项目入口、零容忍规则、快速导航 |
| L2 | L2#规范索引 (Index) | `.ai/L2#Index/` | 分类索引、标题大纲、按需读取导航 |
| L3 | L3#完整规范 (Standards) | `.ai/L3#Standards/` | 当前唯一规范主版本、参考、ADR、案例 |
| L4 | L4#操作日志 (Changelog) | `.ai/L4#Changelog/` | 历史操作记录、迁移记录、决策过程 |
| L5 | L5#知识图谱 (Knowledge) | `.ai/L5#Knowledge/` | 知识沉淀、语义关系、图谱资料 |

## 读取顺序

```
L1#项目概览 (Overview)
  → L2#规范索引 (Index)
  → L3#完整规范 (Standards)
  → L5#知识图谱 (Knowledge)
  → L4#操作日志 (Changelog)
```

L3#完整规范 (Standards) 是当前规范唯一主版本。旧七层架构中的“L3 精炼规范”和“L4 完整规范”已经合并到六层的 L3#完整规范 (Standards)。

## 自动载入层

| 层 | 目标 |
|----|------|
| L1#项目概览 (Overview) | `AGENTS.md`、`CLAUDE.md`、`.cursorrules`、`.windsurfrules` |
| L2#规范索引 (Index) | `.kiro/steering/`、`.cursor/rules/`、`.claude/memories/`、`.continue/rules/`、`.windsurf/rules/` |

L3#完整规范 (Standards) 按需读取，不应整套自动载入到所有上下文。

## 纯化方向

```
L4#操作日志 (Changelog) / L5#知识图谱 (Knowledge)
  → L3#完整规范 (Standards)
  → L2#规范索引 (Index)
  → L1#项目概览 (Overview)
```

L0#工作执行 (Execution) 是执行资产层，不参与规范蒸馏链，但其 hooks/skills/specs 必须遵守 L3#完整规范 (Standards)。

## 零容忍

- 六层物理目录必须使用 `.ai/L#English/` 格式，例如 `.ai/L0#Execution/`。
- 文档中引用层级必须使用中英文混合格式，例如 `L0#工作执行 (Execution)`。
- 新规范只写入 L3#完整规范 (Standards)，不得恢复旧的精炼/完整双规范层。
- L2#规范索引 (Index) 只做导航，不承载规范正文。
- 映射通过 `./gogogo.sh 8f` 或 `./gogogo.sh ai` 管理，不手动维护 IDE 目标目录。
