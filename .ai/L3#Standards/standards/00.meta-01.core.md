# 元规范

> **版本**：v2.0.0（六层架构）
> **最后更新**：2026-04-26

## 文档体系

```
L1#项目概览 (Overview)     ← 项目入口、零容忍规则、快速导航
   ↑
L2#规范索引 (Index)        ← 分类索引、标题大纲、按需读取导航
   ↑
L3#完整规范 (Standards)    ← 当前唯一规范主版本
   ↑
L4#操作日志 (Changelog) / L5#知识图谱 (Knowledge)
```

L0#工作执行 (Execution) 独立存放 hooks、skills、specs、templates、workflows，不参与规范蒸馏链，但必须遵守 L3#完整规范 (Standards)。

## 规范文档命名格式

```
{大类编号}.{类别}-{序号}.{主题}.md
```

| 编号 | 类别 | 说明 |
|------|------|------|
| 00 | `meta` | 元规范 |
| 01 | `arch` | 架构规范 |
| 02 | `backend` | 后端规范 |
| 03 | `frontend` | 前端规范 |
| 04 | `data` | 数据与知识图谱 |
| 05 | `biz` | 业务规范 |
| 06 | `quality` | 质量规范 |
| 07 | `security` | 安全规范 |
| 08 | `ops` / `workflow` | 运维与工作流 |
| 09 | `tool` | 工具规范 |
| 10 | `ref` / `ai-memory` | 参考与 AI 记忆体系 |

**数字含义**：
- 第1段：大类编号（`02` = 后端）
- 第2段：类别内序号（`03` = 第3个主题）
- 第3段+：子主题序号

**举例**：
```
01.arch-01.core.md                  # 架构-第1个-核心
02.backend-03.db-01.tables.md       # 后端-第3个db-子主题01-tables
02.backend-03.db-03.field-02.order.md  # 更深层级
```

## Frontmatter

L2#规范索引 (Index) 和 L3#完整规范 (Standards) 只使用兼容性最高的字段：

```yaml
---
description: "Short rule description"
globs: "**/*.vue"
---
```

`globs` 可选；不要引入工具专属 frontmatter 字段。

## 更新流程

**写入**：

```
L4#操作日志 (Changelog) / L5#知识图谱 (Knowledge)
  → L3#完整规范 (Standards)
  → L2#规范索引 (Index)
  → L1#项目概览 (Overview)
```

**读取**：

```
L1#项目概览 (Overview)
  → L2#规范索引 (Index)
  → L3#完整规范 (Standards)
  → L5#知识图谱 (Knowledge)
  → L4#操作日志 (Changelog)
```

## 禁止

- 恢复旧七层物理目录，如 `.ai/L0/`、`.ai/L3/`、`.ai/L4/`。
- 在 L2#规范索引 (Index) 写规范正文。
- 在 L3#完整规范 (Standards) 之外维护第二套当前规范。
- 新增 `condensed/`、`full/`、`rules/standards/` 作为当前规范目录。
- 新增规范后只改 README，不更新 L2#规范索引 (Index)。
