# 文件命名规范

> 详细规范：[standards/03.frontend-01.structure.md](../standards/03.frontend-01.structure.md)

## 点分命名格式

```
FeatureName.FileType.ext
```

| 类型 | 后缀 | 举例 |
|------|------|------|
| 视图 | `.view.html` | `UsersList.view.html` |
| 逻辑 | `.logic.js` | `UsersList.logic.js` |
| 样式 | `.style.css` | `UsersList.style.css` |
| API | `.api.php` | `UsersList.api.php` |

## FeatureName 生成规则

**路径最后两级 → 大驼峰组合**

| 路径 | FeatureName |
|------|-------------|
| `/admin/users/list/` | `UsersList` |
| `/admin/orders/config/` | `OrdersConfig` |

## 子功能拆分

```
UsersList/
├── UsersList.view.html
├── UsersList.Create.view.html      # 创建
├── UsersList.Edit.view.html        # 编辑
└── UsersList.BatchUpload.view.html # 批量上传
```

## 文件行数限制

| 文件类型 | 推荐 | 上限 | 强制重构 |
|---------|------|------|---------|
| 入口文件 | 20-50 | 80 | >100 |
| 视图文件 | 200-450 | 650 | >800 |
| 逻辑文件 | 200-400 | 600 | >800 |
| 样式文件 | 200-400 | 600 | >1000 |
| 函数/方法 | 10-40 | 80 | >120 |

## 后端类文件

**PHP/Java 禁止点分命名**，使用纯驼峰：

```
❌ UsersList.Controller.php
✅ UsersListController.php
```
