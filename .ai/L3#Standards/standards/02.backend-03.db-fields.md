# 数据库字段命名规范

> 详细规范：[standards/02.backend-03.db-fields.md](../standards/02.backend-03.db-fields.md)

## 字段格式（至少三段式）

字段名**至少**由三段组成，可按需扩展为四段、五段：

```
基础格式：{表级前缀}_{语义分类}_{字段名}
扩展格式：{表级前缀}_{语义分类}_{子分类}_{字段名}
更深层级：{表级前缀}_{语义分类}_{子分类}_{细分}_{字段名}
```

| 段 | 说明 | 举例 |
|---|------|------|
| 表级前缀 | 所属实体 | `base`、`vip`、`itemsimages` |
| 语义分类 | 字段用途 | `core`、`profile`、`image`、`video` |
| 子分类 | 更细分类（可选） | `wechat`、`test` |
| 字段名 | 具体含义 | `name`、`width`、`codec` |

**举例**：
```sql
-- 三段式
base_core_username           -- 基础-核心-用户名
-- 四段式
base_bind_wechat_openid      -- 基础-绑定-微信-OpenID
streamsinfos_video_codec     -- 流信息-视频-编码
-- 五段式
server_speed_test_time       -- 服务器-速度-测试-时间
```

## 常用分类

| 分类 | 说明 | 举例 |
|------|------|------|
| `core` | 核心标识+安全凭证 | `base_core_username` |
| `profile` | 用户自填信息 | `base_profile_nickname` |
| `bind` | 绑定信息 | `base_bind_phone` |
| `status` | 状态标识 | `base_status_active` |
| `count` | 计数统计 | `base_count_login` |

## 字段注释格式

```sql
'序号|中文字段分类|中文字段名|详细说明[|是否显示]'
-- 举例：'1|核心|用户名|用于登录的唯一标识|1'
```

- **序号**：用于排序
- **中文字段分类**：如 `核心`、`资料`、`绑定`
- **中文字段名**：用于前端表格列头
- **详细说明**：枚举值写为 `value1=说明1, value2=说明2`
- **是否显示**：`1`=显示，`0`=隐藏（可选，默认1）

## 例外字段（固定位置）

```sql
id           -- 固定第1位
user_id      -- 固定第2位
created_at   -- 固定倒数第3位
updated_at   -- 固定倒数第2位
deleted_at   -- 固定最后1位
```

## 禁止的命名方式

- ❌ 拼音命名
- ❌ 复合词（如 `emergency_contact`）
- ❌ 暴露实现（如 `_json` 后缀）
- ❌ 旧前缀式（如 `is_`、`num_`）
