---

## 📍 文档定位

本文档定义**路径映射规则**：文件夹路径如何自动映射到数据库表、API、类名、权限点。

---

## 🎯 核心理念

### 路径为王（Path is King）

**文件夹路径是唯一主体**，其他层级全部从路径自动推导：

```
文件夹路径（唯一主体）
pages/admin/users/config/levels/
        ↓ 自动映射
数据库表：admin_users_config_levels
API路径：/api/admin/users/config/levels
后端类名：AdminUsersConfigLevels
权限标识：admin.users.config.levels
```

### 设计原则

- **结构即语义，路径即权限**
- **命名即归属**，不需要额外注释
- 支持**自动识别**和**模块自治**

---

## 🔄 路径映射规则

### 映射转换方法

| 源 | 目标 | 转换方法 | 举例 |
|---|------|---------|------|
| 文件夹路径 | 数据库表 | 下划线连接所有层级 | `admin_users_config_levels` |
| 文件夹路径 | API路径 | 保持层级，加 `/api/` 前缀 | `/api/admin/users/config/levels` |
| 文件夹路径 | 类名 | 各级首字母大写，直接连接 | `AdminUsersConfigLevels` |
| 文件夹路径 | 权限标识 | 点号连接 | `admin.users.config.levels` |

### 完整举例

```
路径：pages/admin/orders/list/
  → 表：admin_orders_list
  → API：/api/admin/orders/list
  → 类：AdminOrdersList
  → 权限：admin.orders.list

路径：pages/home/profile/settings/
  → 表：home_profile_settings
  → API：/api/home/profile/settings
  → 类：HomeProfileSettings
  → 权限：home.profile.settings
```

## 📋 FeatureName 生成规则

**核心公式**：**FeatureName = 路径最后两级目录名组合（大驼峰）**

| 目录路径 | FeatureName |
|----------|-------------|
| `/admin/users/list/` | `UsersList` |
| `/admin/orders/config/` | `OrdersConfig` |
| `/admin/products/detail/` | `ProductsDetail` |

**用途**：用于文件命名，如 `UsersList.view.html`、`UsersList.logic.js`

---

## 📂 职责容器（可选）

当功能复杂时，可使用职责容器区分私有和共享资源：

```
FeatureName/
├── index.html              # 入口
├── FeatureName.view.html
├── FeatureName.logic.js
├── Scope/                  # 私有资源（仅当前功能使用）
│   └── helpers.js
└── Shared/                 # 共享资源（同级及子路径可使用）
    └── utils.js
```

**注意**：`Scope/` 和 `Shared/` 是唯一允许大写的目录名

## 🔄 目录重构规范

### 重构四步流程（强制）

1. **文件移动**到新位置
2. **更新引用**（导入路径、命名空间）
3. **更新路由**配置
4. **测试验证**功能正常

### 检查清单

- [ ] 文件已移动到正确位置
- [ ] 所有导入路径已更新
- [ ] 路由配置已更新
- [ ] 数据库记录已同步（如有）
- [ ] 功能测试通过
