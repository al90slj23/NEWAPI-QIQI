# LibHub 主体设计规范

> 编号: 03.libhub-02 | 分类: LibHub 规范

## 概述

主体（Entity）是指可以拥有独立详情页面、展示自身信息和关联内容的数据实体。

### 主体判定标准

| 条件 | 说明 |
|------|------|
| ✅ 独立页面 | 可以有专属的详情页面 |
| ✅ 自身信息 | 有独立的属性信息（名称、简介、图片等） |
| ✅ 关联内容 | 可以展示与其他主体的关联关系 |
| ✅ 可搜索 | 用户可以直接搜索该主体 |

### 非主体（依附属性）

不满足上述条件的数据作为主体的字段存储：

- **Genres（类型）** - 作为影片的 `itemsinfobase_link_genres` 字段
- **Tags（标签）** - 作为影片的 `itemsinfobase_link_tags` 字段
- **Keywords（关键词）** - 合并到 tags
- **Countries（国家）** - 作为影片的 `itemsinfobase_media_country` 字段

## 当前主体

### 1. Items（影片/剧集）

**核心主体**，所有其他主体都与其关联。

| 页面 | 自身信息 | 关联内容 |
|------|----------|----------|
| 影片详情页 | 标题、简介、评分、海报、年份 | 演员、制作公司、合集、相似影片 |

**表结构**：
```
lab_libhub_storage_items_info_base     # 基础信息
lab_libhub_storage_items_images        # 图片
lab_libhub_storage_items_info_similar  # 相似推荐
lab_libhub_storage_items_info_urls     # 外部链接
```

### 2. Persons（演职人员）

包含演员、导演、编剧等所有人员。

| 页面 | 自身信息 | 关联内容 |
|------|----------|----------|
| 人员详情页 | 姓名、简介、头像、生日、出生地 | 参演/执导影片 |

**表结构**：
```
lab_libhub_storage_persons_infos       # 人员信息
lab_libhub_storage_persons_links       # 人员-影片关联（含角色类型）
lab_libhub_storage_persons_profile     # 头像图片
lab_libhub_storage_persons_urls        # 外部链接
```

**角色区分**：通过 `persons_links.role` 字段区分：
- `Actor` - 演员
- `Director` - 导演
- `Writer` - 编剧
- `Producer` - 制片人

### 3. Studios（制作公司）

| 页面 | 自身信息 | 关联内容 |
|------|----------|----------|
| 公司详情页 | 名称、Logo、国家、简介 | 出品影片 |

**表结构**：
```
lab_libhub_storage_studios_infos       # 公司信息
lab_libhub_storage_studios_links       # 公司-影片关联
lab_libhub_storage_studios_logo        # Logo 图片
```

### 4. Collections（合集）

| 页面 | 自身信息 | 关联内容 |
|------|----------|----------|
| 合集详情页 | 名称、简介、海报 | 包含影片 |

**表结构**：
```
lab_libhub_storage_collections_infos   # 合集信息
lab_libhub_storage_collections_links   # 合集-影片关联
lab_libhub_storage_collections_images  # 合集图片
```

### 5. Networks（播出平台/电视网）

仅用于剧集，存储 HBO、Netflix、ABC 等播出平台。

| 页面 | 自身信息 | 关联内容 |
|------|----------|----------|
| 平台详情页 | 名称、Logo、国家、官网 | 播出剧集 |

**表结构**：
```
lab_libhub_storage_networks_infos      # 平台信息
lab_libhub_storage_networks_links      # 平台-剧集关联
lab_libhub_storage_networks_logo       # 平台Logo
```

**与 Studios 的区别**：
| | Studios（制作公司） | Networks（播出平台） |
|---|---|---|
| **作用** | 负责制作内容 | 负责播出/发行 |
| **适用** | 电影 + 剧集 | 仅剧集 |
| **例子** | Warner Bros, 20th Century Fox | HBO, Netflix, ABC |

## 主体表结构规范

### 标准三表模式

每个主体遵循统一的表结构模式：

```
lab_libhub_storage_{entity}_infos      # 主体信息表（必需）
lab_libhub_storage_{entity}_links      # 主体-影片关联表（必需）
lab_libhub_storage_{entity}_images     # 主体图片表（可选）
lab_libhub_storage_{entity}_urls       # 主体外链表（可选）
```

### 信息表（_infos）必需字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT UNSIGNED | 主键 |
| `tmdb_id` | INT UNSIGNED | TMDB 唯一标识 |
| `{entity}infos_core_name` | VARCHAR | 名称 |
| `{entity}infos_core_overview` | TEXT | 简介 |
| `created_at` | TIMESTAMP | 创建时间 |
| `updated_at` | TIMESTAMP | 更新时间 |

### 关联表（_links）必需字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT UNSIGNED | 主键 |
| `item_id` | BIGINT UNSIGNED | 影片 ID（外键） |
| `{entity}_id` | BIGINT UNSIGNED | 主体 ID（外键） |
| `created_at` | TIMESTAMP | 创建时间 |

### 索引规范

```sql
-- 信息表
UNIQUE INDEX `uk_tmdb_id` (`tmdb_id`)

-- 关联表
INDEX `idx_item_id` (`item_id`)
INDEX `idx_{entity}_id` (`{entity}_id`)
UNIQUE INDEX `uk_item_{entity}` (`item_id`, `{entity}_id`)
```

## 扩展新主体

### 示例：添加 Directors（导演）作为独立主体

如果未来需要将导演从 Persons 中独立出来：

**1. 创建表结构**：
```sql
CREATE TABLE `lab_libhub_storage_directors_infos` (
  `id` bigint unsigned NOT NULL AUTO_INCREMENT,
  `tmdb_id` int unsigned NULL,
  `directorsinfos_core_name` varchar(200) NOT NULL,
  `directorsinfos_core_overview` text,
  `directorsinfos_image_profile` varchar(255),
  `created_at` timestamp DEFAULT CURRENT_TIMESTAMP,
  `updated_at` timestamp DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_tmdb_id` (`tmdb_id`)
);

CREATE TABLE `lab_libhub_storage_directors_links` (
  `id` bigint unsigned NOT NULL AUTO_INCREMENT,
  `item_id` bigint unsigned NOT NULL,
  `director_id` bigint unsigned NOT NULL,
  `created_at` timestamp DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  KEY `idx_item_id` (`item_id`),
  KEY `idx_director_id` (`director_id`),
  UNIQUE KEY `uk_item_director` (`item_id`, `director_id`)
);
```

**2. 添加 API 端点**：
```
GET /lab/libhub/director/{id}  # 导演详情
GET /lab/libhub/directors      # 导演列表
```

**3. 创建前端页面**：
```
msrl/Lab/LibHub/Director/
├── index.html      # 导演详情页
├── director.css
└── director.js
```

## API 设计规范

### 主体详情 API

```
GET /lab/libhub/{entity}/{id}
```

**响应结构**：
```json
{
  "code": 200,
  "data": {
    "info": { /* 主体自身信息 */ },
    "items": [ /* 关联影片列表 */ ],
    "stats": { /* 统计信息 */ }
  }
}
```

### 主体列表 API

```
GET /lab/libhub/{entity}s?page=1&limit=20
```

## 注意事项

1. **TMDB 为主数据源** - 所有主体信息优先从 TMDB 获取
2. **延迟加载** - 主体详情页的关联内容支持分页加载
3. **缓存策略** - 主体信息缓存 24 小时，关联关系缓存 1 小时
4. **向后兼容** - 新增主体不影响现有数据结构
