## 🚀 初始化 Cursor 项目

### 第 1 步：创建 `.cursor` 目录

```bash
cd your-project/
mkdir -p .cursor/rules
```

### 第 2 步：配置 Cursor Rules

在 `.cursor/rules/ald-standards.md` 中创建规范引用：

```markdown
# AiLinkDog 开发规范

## 快速导航

### 优先级 1（必读）
- 规范体系：`../L3#Standards/standards/00.meta-01.core.md`
- 核心架构：`../L3#Standards/standards/01.arch-01.core.md`
- 命名哲学：`../L3#Standards/standards/04.quality-00.naming-philosophy.md`
- API 设计：`../L3#Standards/standards/02.backend-01.api.md`
- 数据库：`../L3#Standards/standards/02.backend-02.db-tables.md`

### 优先级 2（常用）
- Go 模式：`../L3#Standards/standards/02.backend-04.go-patterns.md`
- Channel 集成：`../L3#Standards/standards/02.backend-05.channel.md`
- Relay 架构：`../L3#Standards/standards/02.backend-06.relay.md`
- Git 规范：`../L3#Standards/standards/06.quality-02.git.md`

## 日志记录

完成重要工作后，在 `../L4#Changelog/summaries/` 中创建：
`YYYY-MM-DD-feature-name.md`

参考：`../L3#Standards/standards/07.worklog-01.daily-log.md`
```

### 第 3 步：配置 Cursor 设置

在 `.cursor/settings.json` 中：

```json
{
  "cursor.mode": "standard",
  "cursor.rulesFiles": [
    ".cursor/rules/ald-standards.md"
  ],
  "cursor.defaultModel": "claude-opus-4-6",
  "files.exclude": {
    "**/node_modules": true,
    "**/.git": true,
    "**/pprof": true
  },
  "search.exclude": {
    "**/vendor": true,
    "**/.git": true
  }
}
```

## 💻 Cursor 工作流

### 开发前的准备（5 分钟）

```
1. 打开项目
   Command + Shift + P → "Cursor: Open Folder"

2. 查看规范索引
   Command + K → "@rules ai-index.md"

3. 选择相关规范
   根据任务类型，打开对应的规范文件
```

### 开发中的使用（持续）

#### 使用 Cmd+K 快速查询规范

```
Cmd + K → 输入查询

示例：
"@rules 如何命名变量？" → 返回命名规范
"@rules Channel" → 返回 Channel 集成规范
"@rules Go 模式" → 返回 Go 编码模式
```

#### 在 Tab 补全时应用规范

Cursor 的 Tab 补全会自动参考：
- 打开的规范文件
- `.cursor/rules/` 中的 Prompt
- 项目的命名和架构风格

**最佳实践**
```
边开发边参考规范：
1. 分屏打开规范文件和代码文件
2. 在代码编辑时，Tab 补全会学习规范的风格
3. 遇到疑问，Cmd+K 快速查询
```

#### 代码审查时使用 Cursor

```bash
# 1. 选中代码
# 2. 使用 Cmd+K 提问
Cmd + K → "按照规范审查这段代码"

# Cursor 会参考规范文件进行审查
```

## 🔍 快速查询的最佳方式

### 方式 1：使用 `@rules` 符号

```
Cmd + K → "@rules"

然后输入关键词：
- "@rules 命名" → 所有命名相关规范
- "@rules Go" → Go 相关规范
- "@rules 数据库" → 数据库相关规范
```

### 方式 2：直接引用规范文件

```
Cmd + K → "在 `.ai/L3#Standards/standards/02.backend-04.go-patterns.md`
的基础上，完成以下函数..."

Cursor 会自动读取规范文件并应用
```

### 方式 3：使用分屏显示

```
1. 左侧：规范文件（按 Cmd+\ 分屏）
2. 右侧：代码文件
3. 边参考规范边编码
```

## 📝 Cursor 中的工作日志集成

### 在 Cursor 中记录日志

#### 方式 1：直接创建文件

```bash
# 1. 打开项目文件浏览器
Cmd + E → .ai/L4#Changelog/summaries/

# 2. 新建文件
Cmd + N → 命名为 2026-04-13-feature-name.md

# 3. 使用 Cmd+K 生成模板
Cmd + K → "生成工作日志模板（按照 07.worklog-01.daily-log.md）"

# 4. 填写内容
# 5. 保存并提交
```

#### 方式 2：使用快捷命令

在 `.cursor/rules/` 中添加快捷 Prompt：

```markdown
## 快速创建工作日志

使用此命令：
Cmd + K → "创建今天的工作日志"

Cursor 会：
1. 自动生成日期文件名
2. 插入标准模板
3. 根据今天的提交自动填充内容
```

### 日志的持续记录

**每小时**（15 分钟）
```
快速保存当前进度：
- 完成的小功能
- 遇到的问题
- 临时解决方案
```

**每天下班前**（10 分钟）
```
Cmd + K → "生成今天的每日总结"

Cursor 会汇总：
- 所有完成的功能
- 遇到的问题和解决方案
- 学到的东西
- 明天的计划
```

**每周末**（30 分钟）
```
Cmd + K → "生成本周的周总结"

Cursor 会综合：
- 本周所有的日志
- 关键成果和经验
- 问题分析和改进建议
- 下周计划
```

## ⚙️ Cursor 的高级配置

### 1. 自定义 Tab 补全风格

在 `.cursor/rules/` 中创建 `coding-style.md`：

```markdown
# Cursor Tab 补全风格指南

## Go 代码

遵循：`.ai/L3#Standards/standards/02.backend-04.go-patterns.md`

### 函数命名
- 完整描述 > 缩写
- 例如：ProcessUserRegistration 而非 ProcessReg

### 错误处理
- 始终使用具名返回值
- 例如：func Do() (result int, err error)

## 前端代码

遵循：`.ai/L3#Standards/standards/03.frontend-02.semi-design.md`

### React 组件
- 使用函数组件
- 使用 TypeScript
- 遵循 Semi Design 的命名
```

### 2. 项目级的 AI Prompt

在 `.cursor/rules/` 中创建 `system-prompt.md`：

```markdown
# Cursor 系统 Prompt

你是 AiLinkDog 项目的 AI 开发助手。

## 核心原则

1. 遵循 ZERO 框架规范
2. 参考 ALD 项目特定规范
3. 记录工作日志
4. 提供检查清单

## 规范引用

当用户请求代码时，自动：
1. 查看相关的规范文档
2. 在代码中应用规范要求
3. 提供检查清单
4. 建议记录工作日志

## 工作流

每个开发任务的标准流程：
1. 查看相关规范
2. 生成代码
3. 应用检查清单
4. 提示记录日志
```

## 🎯 典型的 Cursor 工作流

### 场景 1：实现新的 Go 函数

```
1. 打开规范
   Cmd + K → "@rules 02.backend-04.go-patterns.md"

2. 描述任务
   Cmd + K → "实现一个处理用户注册的函数，
   按照规范中的模式"

3. 生成代码
   Cursor 会生成符合规范的代码

4. 应用检查清单
   Cmd + K → "应用检查清单"

5. 记录日志
   新建日志文件，记录完成情况
```

### 场景 2：代码审查

```
1. 选中代码
2. Cmd + K → "按照规范审查这段代码"
3. Cursor 会：
   - 对比规范
   - 指出不符合之处
   - 提供改进建议
4. 应用改进
5. 记录学到的东西
```

### 场景 3：调试问题

```
1. 描述问题
   Cmd + K → "这个错误的原因是什么？"

2. Cursor 会：
   - 查看规范中的错误处理部分
   - 参考项目的最佳实践
   - 提供解决方案

3. 在工作日志中记录：
   - 问题的原因
   - 解决方案
   - 为什么这样解决
   - 下次如何避免
```

## 💡 最佳实践

### 1. 分屏开发

```
左侧：规范文件（不变）
右侧：代码文件（编辑）
底部：终端

这样可以在参考规范的同时编码
```

### 2. 使用快捷键

```
Cmd + K：查询规范或生成代码
Cmd + \：分屏
Cmd + E：打开资源管理器
Cmd + /：打开终端
```

### 3. 每小时保存日志

```
定时保存当前进度：
- 完成的代码
- 待解决的问题
- 学到的东西

这有助于最后写周总结
```

### 4. 参考相关规范

```
不要记住所有规范，而是学会：
- 用 ai-index.md 快速查询
- 用 @rules 快速搜索
- 用分屏参考相关规范
```
