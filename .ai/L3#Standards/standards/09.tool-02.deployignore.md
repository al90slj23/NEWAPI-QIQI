# 09.tool-02.deployignore - 部署排除规则规范

> **文档版本**：v1.0.0
> **创建日期**：2025-12-30

---

## 📍 文档定位

本文档定义 **`.deployignore` 部署排除规则文件**的设计规范。该文件用于配置 rsync 部署时需要排除的文件和目录。

---

## 🎯 核心理念

### 为什么需要 .deployignore

1. **配置与代码分离**：排除规则不硬编码在 gogogo.sh 中
2. **易于维护**：添加/删除排除项只需编辑配置文件
3. **项目可定制**：不同项目可以有不同的排除规则
4. **与 .gitignore 互补**：.gitignore 管 Git，.deployignore 管部署

### 与 .gitignore 的区别

| 文件 | 作用 | 场景 |
|------|------|------|
| `.gitignore` | Git 忽略规则 | 不提交到仓库 |
| `.deployignore` | 部署排除规则 | 不同步到服务器 |

**注意**：两者可能有重叠，但职责不同。例如：
- `vendor/` 可能同时在两个文件中（不提交也不部署）
- `.kiro/` 可能只在 `.deployignore` 中（提交但不部署）

```
project/
├── .gitignore        # Git 忽略规则
├── .deployignore     # 部署排除规则
├── gogogo.sh             # 主入口脚本
└── ...
```

## 📝 文件格式

### 基本语法

```bash
# 注释行（以 # 开头）
.git

# 空行会被忽略

# 目录（不需要尾部斜杠）
vendor
node_modules

# 文件
.env
.DS_Store

# 通配符
*.log
logs/*.log

# 路径模式
media/cache/*
media/posters/*
```

### 语法规则

1. **注释**：以 `#` 开头的行
2. **空行**：自动忽略
3. **目录**：直接写目录名，不需要尾部 `/`
4. **文件**：直接写文件名
5. **通配符**：支持 `*` 通配符
6. **路径**：支持相对路径模式

## 🔧 gogogo.sh 集成

### 读取函数

在 `gogogo.lib.sh` 中添加：

```bash
# ============================================================
# 部署排除规则
# ============================================================

DEPLOY_IGNORE=".deployignore"

# 构建 rsync 排除参数
build_rsync_excludes() {
    local excludes=""
    if [ -f "$DEPLOY_IGNORE" ]; then
        while IFS= read -r line || [ -n "$line" ]; do
            # 跳过空行和注释
            [[ -z "$line" || "$line" =~ ^[[:space:]]*# ]] && continue
            # 去除首尾空格
            line=$(echo "$line" | sed 's/^[[:space:]]*//;s/[[:space:]]*$//')
            [ -n "$line" ] && excludes="$excludes --exclude='$line'"
        done < "$DEPLOY_IGNORE"
    fi
    echo "$excludes"
}

# 预构建排除参数（在脚本加载时执行一次）
RSYNC_EXCLUDES=$(build_rsync_excludes)
```

### 在 gogogo.2.sh 中使用

```bash
# 同步文件到服务器
step "同步文件到服务器..."
eval "rsync -avz --delete --no-owner --no-group --no-perms $RSYNC_EXCLUDES ./ ${REMOTE_USER}@${REMOTE_HOST}:${REMOTE_PATH}/"

if [ $? -ne 0 ]; then
    error "文件同步失败"
    exit 1
fi
success "文件同步完成"
```

## ⚠️ 注意事项

### 1. 不要忽略 .deployignore 本身

`.deployignore` 应该提交到 Git，但可以选择是否部署到服务器。

### 2. 服务器特定配置

服务器上的 `.env` 等配置文件应该手动管理，不要通过部署覆盖。

### 3. 缓存目录

缓存目录（如 `cache/*`）通常只排除内容，保留目录结构：
```bash
cache/*        # 排除内容
!cache/.gitkeep  # 但保留 .gitkeep（如果有）
```

### 4. 测试相关文件

测试框架相关文件通常只在本地使用：
```bash
vendor         # Composer 依赖
tests          # 测试代码
phpunit.xml    # PHPUnit 配置
```
