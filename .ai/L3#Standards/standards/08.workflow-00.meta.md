# 08.workflow-00 - 工作流规范元文档

> **文档版本**：v1.0.0
> **创建日期**：2026-04-12
> **适用范围**：所有 AI 开发平台和工具
> **目标**：将 ZERO 框架和 ALD 规范集成到日常开发工作流

---

## 📋 文档定位

本文档定义 **如何在不同的 AI 开发平台上应用 ZERO 框架和 ALD 规范**，包括：

- 规范体系的快速集成指南
- 各个平台的工作流最佳实践
- 规范文档在工作流中的位置
- 工作日志的自动化记录
- 平台特定的优化建议

---

## 🎯 核心概念

### 规范体系的三个层次

```
第 1 层：ZERO 框架规范（通用）
  ├── 核心架构原则
  ├── 路径映射规则
  ├── 数据库规范
  └── 前端规范
         ↓
第 2 层：ALD 特定规范（项目特定）
  ├── Go 编码模式
  ├── Channel 集成
  ├── Relay 架构
  └── Semi Design UI
         ↓
第 3 层：工作流规范（平台特定）
  ├── Cursor 工作流
  ├── Kiro 工作流
  ├── Claude Code CLI 工作流
  └── OpenCode 工作流
```

### 规范的应用流程

```
新项目启动
    ↓
1. 选择 AI 开发平台
    ↓
2. 加载平台特定的工作流规范
    ↓
3. 集成 ZERO 框架规范
    ↓
4. 集成 ALD（或项目特定）规范
    ↓
5. 配置工作日志记录
    ↓
开始开发
```

## 🔄 工作流与规范的交互

### 典型的开发流程

```
1. 开始新功能
   ↓
2. 查看相关的规范文档
   （从 ai-index.md 快速查询）
   ↓
3. 根据规范编写代码
   ↓
4. 定期记录工作日志
   ↓
5. 完成功能
   ↓
6. 最后提交日志总结
```

### 规范文档的访问方式

**在 Cursor 中**
```
快捷键：Cmd+K （打开命令面板）
输入：@rules ai-index.md （快速查询）
```

**在 Kiro 中**
```
左侧面板：点击 Rules 标签
搜索：输入功能名称
快速查询表：自动显示相关规范
```

**在命令行中**
```bash
cat .ai/L3#Standards/standards/02.backend-04.go-patterns.md
grep -r "Channel" .ai/L3#Standards/standards/
```

## 💡 规范与工作日志的整合

### 工作日志中如何引用规范

```markdown
# 2026-04-13 - 实现新的 Channel

> **相关规范**：[02.backend-05.channel.md](../L3#Standards/standards/02.backend-05.channel.md)

## 完成内容

- 按照规范实现了新的 Channel 适配器
- 遵循了 Go 编码模式规范
- 完成了错误处理的检查清单
```

### 工作日志推动规范改进

```
开发过程
    ↓
遇到规范不清楚的地方
    ↓
在工作日志中记录疑问
    ↓
定期审查工作日志
    ↓
改进相关的规范文档
    ↓
更新 ZERO 框架
```

## 🎯 推荐的学习路径

### 第 1 天：快速入门

```
1. 读本文档（15 分钟）
2. 选择工作流规范（5 分钟）
3. 配置 IDE（30 分钟）
4. 读 P1 规范（38 分钟）
   → 现在可以开始编码了！
```

### 第 1 周：建立习惯

```
每天：
- 在开始新任务时查规范（5 分钟）
- 编写代码（大部分时间）
- 完成时记录日志（10 分钟）

周末：
- 回顾本周的工作日志
- 总结学到的东西
- 识别规范需要改进的地方
```

### 第 1 个月：掌握完整体系

```
- 深入学习 P2 和 P3 规范
- 参与规范的改进和优化
- 建立个人的最佳实践文档
- 与团队分享经验
```
