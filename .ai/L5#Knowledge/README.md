# L5#知识图谱 (Knowledge) — 知识图谱（可选扩展）

**定位**：语义搜索引擎，碎片化知识存储，向量嵌入，跨会话记忆。

## 核心原则

- **越大越好** — 知识积累越丰富，搜索越精准
- **语义搜索** — 通过向量相似度精准获取相关知识片段
- **节省 Token** — 无需读取完整文档，只获取相关片段（节省 90%+）
- **跨会话记忆** — 保留历史架构决策和开发经验

## 技术选型

| 技术 | 说明 |
|------|------|
| **FalkorDB** | 高性能图数据库（推荐） |
| **Neo4j** | 成熟的图数据库 |
| **其他** | 任何支持向量搜索的图数据库 |

## 访问方式

通过 MCP 工具访问，例如：
- Graphiti MCP（FalkorDB）
- Neo4j MCP
- 自定义 MCP 服务器

## 集成流程

```
L3#完整规范 (Standards)/L4#操作日志 (Changelog) 内容
    ↓ 提取关键信息
向量嵌入（如 text-embedding-3-small）
    ↓ 存储到图数据库
L5#知识图谱 (Knowledge) 知识图谱
    ↓ 语义搜索
通过 MCP 工具查询
```

## 使用场景

1. **精炼规范不够详细时**
   - L2#规范索引 (Index) 提供核心要点
   - L5#知识图谱 (Knowledge) 提供相关历史经验和详细片段

2. **查找历史决策或经验时**
   - 无需翻阅完整 L4#操作日志 (Changelog) 日志
   - 通过语义搜索快速定位

3. **跨文档关联信息时**
   - 自动关联相关规范和实践
   - 发现隐藏的知识联系

## 核心价值

### 1. 节省 Token（90%+）
```
传统方式：读取完整 L3#完整规范 (Standards) 文档（5000+ tokens）
L5#知识图谱 (Knowledge) 方式：只获取相关片段（500 tokens）
节省：90%
```

### 2. 精准获取
```
关键词搜索：可能遗漏相关内容
语义搜索：理解意图，精准匹配
```

### 3. 历史积累
```
随着开发推进：
- 知识图谱越来越丰富
- 搜索结果越来越精准
- AI 开发效率越来越高
```

## 注意事项

1. **L5#知识图谱 (Knowledge) 是可选的**
   - 小型项目可以只使用 L0#工作执行 (Execution) - L4#操作日志 (Changelog)
   - 大型项目建议使用 L5#知识图谱 (Knowledge) 提升效率

2. **需要额外配置**
   - 图数据库服务器
   - MCP 工具
   - 向量嵌入模型

3. **持续维护**
   - 定期从 L3#完整规范 (Standards)/L4#操作日志 (Changelog) 提取新知识
   - 清理过时或错误的信息
   - 优化搜索质量
