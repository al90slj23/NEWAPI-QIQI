## L2 定位

**L2 = 分类索引 + 文件标题大纲**

### 为什么需要文件标题大纲？

传统的索引只有文件名和简短描述，AI 无法判断文件是否包含所需内容，容易：
- ❌ 盲目加载多个不相关文件
- ❌ 浪费大量 token
- ❌ 降低响应速度

通过提供每个文件的标题大纲（h1/h2/h3），AI 可以：
- ✅ 快速判断文件是否包含所需内容
- ✅ 精准定位，只加载 1-2 个相关文件
- ✅ 减少 token 消耗 40-50%
- ✅ 提高响应速度

## 维护指南

### 何时更新

1. **新增 L3#完整规范 (Standards) 文件**：必须更新索引
2. **L3#完整规范 (Standards) 文件标题变更**：必须更新大纲
3. **文件分类调整**：必须更新索引结构

### 更新流程

1. 修改 `toc.md`
2. 添加或更新文件索引
3. 添加或更新文件标题大纲
4. 确保行数在 1200 行以内（硬限制 1500 行）
5. 更新"最后更新"日期

### 质量检查

```bash
# 检查行数
wc -l ".ai/L2#Index/toc.md"

# 检查是否包含所有 L3 文件
find ".ai/L3#Standards/standards" -name "*.md" | wc -l
grep -c "L3#Standards" ".ai/L2#Index/toc.md"

# 检查索引完整性
# 确保所有 L3 文件都在 L2 中有索引
```
